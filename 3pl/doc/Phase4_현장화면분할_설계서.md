# PackFlow 현장용(Field) 시스템 화면 분할 설계서 (통합본)

> **작성 목적**: 작업자의 역할, 동선, 하드웨어 환경이 전혀 다른 현장 특성을 반영하여, 에러를 방지하고 처리 속도를 극대화하기 위한 현장용 UI/UX 분할 아키텍처 정의.
> **AI 리뷰**: Gemini 보강안 반영

---

## 1. 분할 설계의 필요성 (핵심 원칙)

* **하드웨어의 파편화**: 고정된 PC와 웹캠을 쓰는 '포장 작업자'와 PDA를 들고 창고를 누비는 '재고 관리자'의 화면은 같을 수 없습니다.
* **휴먼 에러 원천 차단**: 하나의 통합 화면에 모든 기능을 넣으면, 포장 아르바이트생의 조작 실수(화면 터치, 클릭 오작동)로 대규모 재고 데이터가 훼손될 위험이 큽니다.
* **No-Click UI 구현**: 작업자는 장갑을 끼고 물건을 양손에 든 상태입니다. 각 화면은 마우스 클릭 없이 **'바코드 스캔'**만으로 모든 상태 전이가 이루어져야 합니다.

---

## 2. 현장용 시스템 3대 분할 모듈

### 모듈 A: 패킹 스테이션 뷰 (포장/검수/출고)

가장 빠르고 정확하게 출고를 마무리지어야 하는 **고정형 작업대** 전용 화면입니다.

* **사용 장비**: 데스크탑 PC (대형 모니터), 유선 바코드 스캐너, 고해상도 웹캠, 택배 송장 프린터
* **핵심 목적**: 오배송률 0% 달성, 클레임 방어용 영상 증빙, 출고 과금 트리거
* **UI/UX 특징**:
  * **초대형 시각 피드백**: 모니터 멀리서도 보이도록 상품 이미지와 [정상(녹색) / 수량부족(황색) / 오배송(적색)] 상태를 화면 전체에 띄웁니다.
  * **자동화 플로우**:
    1. 송장(또는 바구니) 바코드 스캔 → 웹캠 녹화 자동 시작
    2. 상품 바코드 스캔 → 수량 충족 시 화면 전환음(비프음) 발생
    3. **웹캠 녹화 자동 종료 및 클라우드 업로드**
    4. **송장 자동 출력** 및 DB상 '출고 완료(과금 대상)' 처리

### 모듈 B: 모바일 PDA 뷰 (재고 및 로케이션 관리)

창고 전체를 돌아다니며 실물 재고와 DB 데이터를 100% 동기화하는 **물류 관리자** 전용 화면입니다.

* **사용 장비**: 안드로이드 기반 산업용 PDA, 스마트폰, 태블릿 PC
* **핵심 목적**: 입출고 적치(Put-away), 로케이션(위치) 간 재고 이동, 실사 및 불량 차감
* **UI/UX 특징**:
  * **위치(Location) 중심 설계**: 상품 자체보다 해당 상품이 '어느 구역, 몇 번 랙'에 있는지 정보가 최우선으로 노출되어야 합니다.
  * **직관적인 강제 조정**:
    1. 특정 랙(Rack) 바코드 스캔 → 해당 위치의 시스템상 재고량 출력
    2. 실물과 다를 경우, PDA 상에서 즉시 사유(파손, 분실, 화주 요청) 입력 후 재고 증감 처리

### 모듈 C: 피킹 내비게이터 뷰 (집품 및 동선 최적화)

가장 짧은 동선으로 대차(카트)를 끌고 다니며 물건을 쓸어 담는 **피커(Picker)** 전용 화면입니다.

* **사용 장비**: 대차에 거치된 태블릿 PC 또는 스마트폰 + 무선 링 스캐너(손가락 착용)
* **핵심 목적**: 작업자의 이동 거리 최소화 (UPH 극대화)
* **UI/UX 특징**:
  * **동선 알고리즘 적용**: 주문이 들어온 순서가 아니라, 창고 도면상 '가장 가까운 로케이션 순서대로' 픽업 리스트를 재정렬하여 화면에 띄웁니다.
  * **지시형 인터페이스**:
    * 화면 노출 예: `"다음 목적지: [A존-03열-1단] 이동하여 → [신라면 50개] 집으세요."`
    * 수량 확인 후 완료 버튼 터치 시 다음 최단 거리 로케이션 지시.

## 3. 권한 및 접근 제어 (RBAC)

각 현장 작업자는 본인에게 부여된 고유 계정으로 로그인하며, 시스템은 계정 권한에 따라 화면을 철저히 통제합니다.

| 권한 레벨 | 허용 모듈 | 차단 모듈 |
|-----------|----------|----------|
| **단기 알바 (패커)** | 모듈 A (패킹 스테이션) | 모듈 B (재고 조정 접근 불가) |
| **현장 사원 (피커)** | 모듈 C (피킹 내비게이터) | 모듈 A, B 접근 불가 |
| **물류 팀장 (관리자)** | 모듈 A, B, C 전체 접근 | 없음 |

## 4. GPT 리뷰 반영 — 작업 모드 기반 5분할

> "사람은 나뉘고, 화면은 나뉘지만, 데이터와 시스템은 하나다"

### 4-1. 핵심 원칙

```
시스템은 하나, 화면은 분리
  - DB / API / 상태흐름 = 하나
  - UI만 역할별로 다르게 제공
  - 기능이 아니라 "작업 모드"로 분리
```

### 4-2. 5개 작업 모드 (Gemini 3모듈 + GPT 2모듈 추가)

```
현장 포털 (단일 시스템)
  ├── 입고/재고 모드     ← Gemini 모듈B 확장
  │     입고스캔, 로케이션지정, 재고조회, 실사, 조정
  │
  ├── 피킹 모드          ← Gemini 모듈C
  │     피킹리스트, SKU별 총수량, 동선최적화
  │
  ├── 송장확인 모드      ← GPT 추가 (사전 검증)
  │     송장출력목록, 취소/보류 확인, 오류 사전차단
  │
  ├── 출고/상차 모드     ← Gemini 모듈A 분리
  │     송장스캔, 출고확정, 상차완료
  │
  └── 촬영/검수 모드     ← Gemini 모듈A 분리
        주문스캔, 바코드검수, 영상촬영, 클라우드저장
```

### 4-3. 권한 매핑 (통합)

| 역할 | 입고/재고 | 피킹 | 송장확인 | 출고/상차 | 촬영/검수 |
|------|:--------:|:----:|:-------:|:--------:|:--------:|
| 단기알바(패커) | - | - | - | - | ✅ |
| 피커 | - | ✅ | ✅ | - | - |
| 입고담당 | ✅ | - | - | - | - |
| 출고담당 | - | - | ✅ | ✅ | ✅ |
| 물류팀장 | ✅ | ✅ | ✅ | ✅ | ✅ |

### 4-4. UI 설계 원칙 (모드별)

| 모드 | UI 특성 | 핵심 |
|------|---------|------|
| 입고/재고 | 테이블 중심, 수량 입력 강조 | 정확도 |
| 피킹 | 큰 버튼, 최소 클릭, 속도 최적화 | 속도 |
| 송장확인 | 상태 색상 강조 (정상/취소/보류) | 사전 검증 |
| 출고/상차 | 스캔 중심, 즉시 피드백 | 오류 방지 |
| 촬영/검수 | 카메라 중심, 검수 상태 표시 | 증빙 |

### 4-5. 운영 주의사항

- **너무 세분화 금지**: 한 사람이 여러 역할 수행 가능해야 함
- **모드 구분 명확**: 색상/UI 차이로 직관적 구분
- **상태는 하나로 유지**: 화면별 상태 따로 만들면 데이터 꼬임

## 6. 현재 3PL 구현 상태 vs 설계

### 이미 구현된 것

| 모듈 | 현재 구현 | 경로 |
|------|----------|------|
| **모듈 A 패킹** | ✅ 촬영모드 + 속도모드 + 바코드 스캔 | `/packing/recording`, `/packing/speed` |
| **모듈 B PDA** | ✅ 현장모드 (입고/이동/실사/출고) | `/packing/field/*` |
| **모듈 C 피킹** | ✅ 피킹리스트 + 스캔 확인 | `/packing/picking` |

### 개선 필요 사항

| 항목 | 현재 | 설계 목표 |
|------|------|----------|
| **화면 분할** | 하나의 패킹 포털에 모든 기능 | 역할별 진입점 분리 (로그인 시 자동 라우팅) |
| **No-Click UI** | 일부 클릭 필요 | 바코드 스캔만으로 전체 플로우 |
| **시각 피드백** | 일반 웹 UI | 대형 모니터용 풀스크린 상태 표시 |
| **동선 최적화** | 미구현 | 로케이션 좌표 기반 최단 경로 정렬 |
| **RBAC 분리** | 부분 적용 | 역할별 메뉴 완전 격리 |

## 7. 구현 계획

### Phase 1: 역할별 진입점 분리
- 로그인 시 role 기반 자동 라우팅
  - `packing_worker` → 모듈 A (패킹 스테이션)
  - `warehouse` → 모듈 B (PDA 뷰)
  - `packing_lead` → 모듈 C (피킹) + 관리자 전체 접근
- 각 모듈별 독립 레이아웃 (base_packing.html, base_pda.html, base_picking.html)

### Phase 2: No-Click UI 고도화
- 바코드 스캔 이벤트 핸들러 통합
- 액션 바코드 (작업완료, 에러무시, 다음건) 인쇄하여 작업대 부착
- 청각 피드백: 정상(경쾌음), 경고(경고음), 에러(요란한 경고)

### Phase 3: 피킹 동선 최적화
- warehouse_locations 테이블에 좌표(x, y, floor) 칼럼 추가
- 피킹리스트 생성 시 최단 경로 정렬 (TSP 근사 알고리즘)
- 피킹 네비게이터 UI (다음 목적지 + 남은 품목수)

### Phase 4: 대형 모니터 풀스크린 UI
- 패킹 스테이션 전용 풀스크린 모드
- 상태별 배경색 전환 (녹/황/적)
- 상품 이미지 대형 표시
- 수량 카운터 대형 숫자
